# Reglas

Reglas de trabajo del proyecto. La fuente de verdad es [../../CLAUDE.md](../../CLAUDE.md) (se carga automáticamente en cada sesión).

Resumen actual:
1. Verificar el código de verdad antes de reportar cualquier estado.
2. No inventar bugs, riesgos de seguridad u optimizaciones que no estén confirmados.
3. Solo avanzar de fase con: sin bugs, código optimizado, sin problemas de seguridad, y test end-to-end exitoso.
4. Antes de cerrar una fase, correr un test de seguridad específico sobre lo modificado en esa fase (no solo que funcione — que siga siendo seguro).
5. El usuario puede agregar más reglas en cualquier momento.
